# 블로그 프로젝트 시작 🚀

**개인 블로그를 직접 만드는 과정**을 다뤄볼 예정입니다. 추후 아래에 글 추가 예정

## 동기 💪
- 이번에 개발 공부를 시작하면서 매일 TIL을 작성하고 있는데 노션에 작성하고 팀원들한테 공유하는 방식으로 작성 중인데 꽤나 열심히 쓰고 있어서 조금 아까운 마음에 블로그에 올리고 싶다는 생각을 하게 되었다.
- github 페이지로 배포해서 진행하려하다가 (jekyll theme활용) 템플릿이 다 썩 마음에 안드는 것이다.
- 그것도 그건데 정작 풀스택을 공부하면서, 물론 아직 프론트엔드만 조금 건들이는 수준이지만 간단한 형태의 읽기 전용 블로그 정도는 응당 직접 만들어서 운영해야하지 않을까 해서 시작하게 되었다.

## 목표 🎯
- 리액트 (Next.js)에 익숙해지기
- 지난 프로젝트에서 구현 못해본 다크모드, 무한 스크롤 등 다양한 기능 구현해보기
- 검색엔진 최적화 등 안해본 기술 사용하기
- 한번에 완성하지 않아도 되니 일단 눈 버릴 정도 아니면 배포해서 운영하면서 수정하기

## 사용 기술 🛠
- **Next.js**: 정적 사이트 생성을 위한 프레임워크
- **MDX**: Markdown과 React 컴포넌트 결합
- ~~**Tailwind CSS**~~: 사용하지 않음, 지난 프로젝트에서 경험해본 결과 섬세한 스타일링이 부족하게 느껴졌음.

## 요구사항 🙋‍♀️
#### 최소 기능
- 블로그 목적은 TIL 및 공부한 내용을 쉽게 찾아보기 위함이다.
		- 즉, 인라인 코드, 코드 블럭, aside, 테이블 등 mdx 스타일링이 잘 되어야 한다.
- TIL은 월 별 필터 기능이 필요하다.
- Study
		- 기술 스택 별로 카테고리 필터링이 필요하다.
		- 제목 및 한줄 설명 필터로 검색 기능이 필요하다.
- 전체 다크모드 기능

> 🚧 앞으로 기능을 하나씩 추가하며 성장하는 프로젝트가 되었으면 좋겠다.
