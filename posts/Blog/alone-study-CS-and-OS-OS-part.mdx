## 운영체제 시작하기

### `운영체제`

- 실행할 프로그램에 필요한 `자원` 을 할당하고, 프로그램이 올바르게 실행되도록 돕는 특별한 프로그램
    - 자원 - CPU, 메모리, 보조기억장치, 입출력 장치 등 컴퓨터 부품들
- 컴퓨터가 부팅될 때 메모리 내 `커널 영역` 에 따로 적재되어 사용자 영역에 적재된 프로그램에 자원을 할당하고 실행되도록 돕는다.
    - 나머지 영역은 `사용자 영역`
- 지속적인 메모리 자원 관리
    - 사용자 영역의 메모리 주소가 겹치지 않도록 적당한 공간에 적재
    - 실행되지 않는 프로그램을 메모리에서 삭제
    - 공정하게 여러 프로그램에 CPU 자원 할당

### 운영체제 왜 알아야 하는가?

- 하드웨어 상태가 어떤지, 코드가 어떻게 실행 되었는지, 하드웨어 상 어떤 문제가 있었는지 등을 알려 준다. ⇒ 문제 해결의 실마리를 제공
- 하드웨어와 프로그램을 더 깊이 이해할 수 있음
- 운영체제와의 대화 예시 - 오류 메시지

### 운영체제의 큰 그림

**`커널` -** 운영체제의 핵심 서비스 (자원에 접근하고 조작하는 기능, 프로그램이 올바르고 안전하게 실행되게 하는 기능)

- UI는 커널에 포함되지 않음
    - GUI - 그래픽 유저 인터페이스
    - CLI - 커맨드 라인 인터페이스

- 운영체제는 응용 프로그램 대신 자원에 접근하여 요청한 작업을 수행
- **`이중모드`**
    - CPU가 명령어를 실행하는 모드를 크게 사용자 모드와 커널 모드로 구분하는 방식
    - `사용자 모드`
        - 운영체제 서비스를 제공받을 수 없는 실행 모드
        - 자원에 접근 불가
    - `커널 모드`
        - 커널 영역의 코드를 실행할 수 있는 모드
        - 자원에 접근하는 명령어 + 모든 명령어 실행 가능
    - 어떤 모드인지 확인은 플래그 레지스터 속 슈퍼바이저 플래그 확인
- `시스템 호출`
    - 사용자 모드에서 실행되는 프로그램이 자원에 접근하는 운영체제 서비스를 제공받기 위한 요청
    - 시스템 호출을 통해 커널 모드로 전환해서 운영체제 서비스를 제공 받을 수 있음
    - `소프트웨어 인터럽트` 의 일종
- `운영체제의 핵심 서비스`
    - `프로세스 관리`
        - CPU는 한 번에 하나의 프로세스만 실행하기에 번갈아 가며 실행함
        - 이를 관리하고 실행함
    - `자원 접근 및 할당`
        - CPU
            - `CPU 스케줄링` - 프로세스들에 공정하게 CPU를 할당할지 결정
        - 메모리
            - 새로운 프로세스가 적재될 때마다 어느 주소에 적재할지 결정
        - 입출력 장치
            - 커널영역의 인터럽트 서비스 루틴을 제공
    - `파일 시스템 관리`
        - 보조 기억장치 속 데이터를 파일과 디렉터리로 관리
