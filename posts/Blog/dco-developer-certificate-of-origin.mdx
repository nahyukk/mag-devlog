- 커밋 메시지에 "Signed-off-by" 문구를 추가해서 DCO에 동의를 하고 기여를 인증하는 것

#### DCO 서명 적용하기
방법은 간단한데 커밋할 때 아래와 같이 하면 됨.

```coq
git commit -s -m “메시지 메시지”
```

근데 문제는 여기서 발생했다. 깃허브 이름을 귀엽게 작성해놨는데 조금 부끄러워서 그냥 본명을 쓰려고 프로필을 수정했지만 반영되지 않았다. 
(사실 아직 PR 날리기 전이었고 지우고 다시 클론 받아서 했는데도 변하지 않았다. 시간이 필요한가…? 로컬에서도 설정이 필요한지..) 
이미 벌어진 일이니 수정하는 것으로 마음 먹었다.

#### DOC 이름 변경하기

이름 수정

```coq
git config user.name "내 이름"
```
<br />
이름 덮어쓰기

```coq
git commit --amend --reset-author -s
```
<br />
<a href="https://drive.google.com/file/d/185Q_hYXgVpBsdGlF3d9MgdRmGoind8mp/view?usp=drive_link" target="_blank">
  <img src="https://lh3.googleusercontent.com/d/185Q_hYXgVpBsdGlF3d9MgdRmGoind8mp" width="100%" />
</a>

- 주의 할 점은 이름 덮어쓰기를 하게 되면 Signed-off-by 는 하나 더 생기게 된다. 이 부분은 직접 에디터로 들어가서 지워줘야 함.

```coq
git commit --amend
```

- vi 에디터라서 insert해서 지우고 :wq 해서 저장하고 나오면 됨.
    - 가끔 저장 안되고 에러 나기도 하는데 그냥 다시 들어가서 수정하고 저장하면 됨.
- 여기서 Commit 한 사람 이름도 수정 가능함.
<br />
- 아래 코드로 확인

```coq
git log -1 --pretty=full
```
